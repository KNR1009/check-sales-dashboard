# 営業管理ダッシュボード化システム　要件定義書（ドラフト v0.1）

作成日：2026年6月23日
作成：株式会社サードスコープ
対象：不動産売買・賃貸管理事業（買い切り再販／仲介、8月以降は福祉事業も視野）

---

## 0. この文書の位置づけと前提

本書は、先日の打ち合わせで出たアイデアを「全部詰め込む」のではなく、**優先度（Phase）をつけて段階的に実現する**ための要件定義の初版です。

打ち合わせでも合意した通り、データはこれから設計し、アンケートフォーム（Googleフォーム）や日報で取得していきます。したがって本書はまず**「理想形（あるべき姿）」を定義**し、そこから逆算して「最初に何を作るか・どのデータを取るか」を決める構成にしています。

重要な前提認識：

- **AIやダッシュボードの前に、まず定量データが存在しないと何も始まらない。** 現状は結果（契約したか／していないか）のデータはあるが、その手前の行動量（テレアポ・訪問・DM何件か）が数値で残っていない。
- したがって**最優先は「定量データを取得できる仕組みづくり」**であり、ダッシュボードはその上に乗るもの。
- 過去にSalesforceを導入したが「高コスト」「全員が使いこなせない」で費用対効果が合わず撤退した経緯がある。今回は**自社で内製（Claude／GAS／Googleフォーム等）し、現場全員が使える簡素さ**を最優先方針とする。

## 1. 背景と課題（As-Is）

### 1.1 事業・組織の状況

- 不動産売買（買い切り再販・仲介）が本業。少人数体制。
- 2026年5月より売買業務を**インセンティブ制**に移行。
- 8月にM&Aで賃貸管理（および福祉事業）が加わり、手元資金が厚くなる見込み。AI・営業領域への投資を強化したい意向。
- 中長期では「AIネイティブカンパニー」への移行を目指す。

### 1.2 現状の管理方法と課題

| 領域 | 現状 | 課題 |
|---|---|---|
| 顧客管理 | スプレッドシートで「誰が・どの物件・どの顧客担当か」を管理 | 履歴が残っていない。担当ベースで属人化 |
| 成約管理 | 契約シート（担当・契約日・決済日／入金日）。買付が入った時点で入力 | 成約“結果”のみ。手前の営業プロセスが記録されない |
| 営業活動記録 | 週1回の口頭ミーティングで共有 | 記録化されていない。後から分析できない |
| 日報 | チャットワークで毎日提出（感想・所感ベース） | 定量データがない（「何件訪問したか」が分からない）。チャットワークはMCP未公開でAI連携不可。提出は責任者クラス中心で一般営業は徹底されていない |
| インセンティブ | 入出金シートに数値はある | 個人別の到達額が見える化されていない |
| 物件概要書 | スプレッドシートで作成。レントロール等は月次で作り直し | 変動箇所（家賃・入居状況・固定資産評価）以外も毎回作り直しており非効率 |
| 各種シート | 売上シート、原価管理シート（買い切り再販）等が分散 | データが分断され、横断的に見られない |

### 1.3 解決したい本質的な問い

打ち合わせで繰り返し出た、経営として可視化したい論点：

1. **「そもそも行動しているのか」** … 活動量が少ないのに定時で帰り、結果が出なくても不満を言う層を炙り出したい。
2. **「行動の割に成果が出ているか」** … 活動量と成果の相関を見て、リスト・トーク・進め方のどこに問題があるかを特定したい。
3. **「個人ごとに何を改善すべきか」** … 数値の可視化だけでなく、「あなたはここができていない、こうすべき」というアドバイスまで出したい。

## 2. システムの目的とコンセプト（To-Be）

### 2.1 目的

営業の**行動量と成果を定量的に見える化**し、

1. 経営・マネジメントが「炙り出し」と「公平な評価（インセンティブ）」を行えるようにする
2. 各営業に対し「次に何をすべきか」のネクストアクションと改善アドバイスを提示する
3. 人がやらなくてよい作業（資料作成・転記・配信）をAIに寄せ、人は対面の商談（残り1〜2割）に集中できるようにする

### 2.2 設計原則

- **最小限主義**：あらゆるデータを一画面に出すと逆に見づらくなる。「成果が良くない」「行動量が足りない」が一目で分かる最小限の指標に絞る。
- **定量ファースト**：所感ではなく数値で記録・比較する。AIはその上に乗せる。
- **全員が使える簡素さ**：Salesforceの失敗を繰り返さない。入力負荷を極小化し、強制力のある運用とセットで設計する。
- **人とAIの役割分担を明確化**：8〜9割をAI／仕組みで処理し、残りの人的リソースを高付加価値業務に集中させる。
- **内製・段階導入**：Claude／GAS／Googleフォーム等で小さく作り、現場の声を取り込みながら拡張する。

### 2.3 全体イメージ（データの流れ）

```
[入力層]
 Googleフォーム（日報・営業活動）  契約/入出金シート  物件データ
        |                              |              |
        v                              v              v
[データ統合層]  Googleスプレッドシート（正規化された蓄積先）
        |
        v
[分析・AI層]  Claude / GAS  ＋ 評価マニュアル（考え方）を裏側に組込み
        |
        v
[出力層]  ダッシュボード（行動量×成果の可視化＋AI考察レポート＋ネクストアクション）
```

## 3. ダッシュボードに表示する指標（中核要件）

打ち合わせで「最重要」として合意した4軸を中心に構成する。

### 3.1 軸①：行動量（活動量）

何件、どんな営業活動をしたか。営業手法に合わせて記録する。

- テレアポ件数 / 訪問（アポ取得→訪問）件数 / 銀行訪問件数
- 空き家調査・DM発送件数
- そのうち「有効リード（ニーズがありそう）」だった件数

→ 目的：「そもそも活動していない」を明示的に見せる。

### 3.2 軸②：成果（契約・売上）

- 契約件数、契約日、決済日（入金完了日）
- 売上額、（買い切り再販の場合）原価・粗利
- パイプライン状況（買付前〜決済までのステータス）

→ 目的：結果を公平に可視化する。インセンティブと連動。

### 3.3 軸③：行動量 × 成果の相関 ★最重要

- 「行動量は多いが成約が少ない」→ リスト・トーク・進め方に問題の可能性
- 「行動量も成果も少ない」→ そもそもの活動不足
- 「行動量は少ないが成約は出ている」→ 効率型

→ 目的：佐々木さんが最も実現したい「行動の割に成果が出ていない／そもそも動いていない、を全員の前で示せる」状態をつくる。散布図やランキング、対前週比などで表現。

### 3.4 軸④：AI考察・改善アドバイス

- あらかじめ作成済みの**1ページの評価マニュアル（考え方）**を裏側に組み込む。
- 個人ごとの定量データと照合し、「あなたは行動量が不足している」「成約率が低い、トークのこの部分が弱い」といった**考察レポート**を自動生成。
- 単なる可視化ではなく「マニュアルに書かれていることができていない」というアドバイスベースの出力にする。
- 「返信のない顧客にはアプローチすべき」等、**ネクストアクションの提示**まで行う。

### 3.5 インセンティブ可視化（軸②の派生）

- 入出金シートを読み込み、「Aさん：◯◯万円／Bさん：◯◯万円」と個人別の到達インセンティブを見える化。

### 3.6 ダッシュボード設計上の注意

- 統合しすぎない。まずは上記の最小限から。
- 「成果が良くない人」「行動量が足りない人」がパッと分かるUIを優先（ランキング・色分け・アラート）。
- 個人ビュー（自分の改善点）とマネジメントビュー（全員比較）を分ける。

## 4. 優先度付きスコープ（Phase 計画）

打ち合わせで出た全アイデアを Phase に振り分けた。**Phase 0 が成立しないと Phase 2 以降は成立しない**点が最重要。

### Phase 0  — 定量データ取得の仕組みづくり【最優先・前提条件】

> 「ダッシュボードを作る前に、まず定量データを取れるようにする」

- **日報をチャットワークからGoogleフォームへ移行**（チャットワークはMCP未公開でAI連携不可のため）。
- 日報フォームを定量項目中心に再設計：本日の訪問件数／テレアポ件数／DM件数／有効リード件数 等。所感は最小限に。
- 全営業（責任者だけでなく一般営業も）に**強制力をもって**入力させる運用ルールを定める。
- 取得データの蓄積先スプレッドシートを設計（後続の連携を見据えた正規化）。

成果物：日報Googleフォーム（設計案）、入力項目定義、運用ルール案。

### Phase 1 — 既存データでダッシュボード試作（成果・インセンティブの可視化）

> 「今あるデータだけで、まずイメージを形にする」

- 既存の契約シート・入出金シートを連携し、成果（契約・決済・売上）と個人別インセンティブを可視化。
- サンプルUI/UX（ダッシュボードのたたき台）を作成し、佐々木さんに確認いただく。
- ここで「こういうデータも欲しい」という追加要望を吸い上げ、Phase 0 のフォーム設計に反映（逆算設計）。

成果物：ダッシュボード試作（成果＋インセンティブ）、UI/UXサンプル。
※サンプルデータの提供が前提（秘密保持契約を締結のうえ受領）。

### Phase 2 — 行動量データ統合・相関分析・AI考察【本命】

> 「行動量×成果」「AIアドバイス」を実装する中核フェーズ

- Phase 0 で取得し始めた行動量データを統合。
- 軸③（行動量×成果の相関）を実装。
- 軸④（評価マニュアルを組み込んだAI考察レポート・ネクストアクション）を実装。
- 営業記録（週1口頭MTGの内容）も記録化してデータ化。

成果物：完全版ダッシュボード（4軸）、AI考察レポート機能。

### Phase 3 — 周辺業務の自動化・高度化【将来構想】

> やりたいことは多いが、ダッシュボード安定後に着手

- **物件概要書／レントロールの半自動化**：変動箇所（現在の家賃・入居状況・固定資産評価）だけ入力すれば出力されるテンプレート化。
- **メール自動配信**：ダッシュボードで「未アプローチ」と判定された顧客への自動メール。
- **営業MTGの音声分析**：録音ツール（TLDV等）から商談・会議を文字起こしし、「このMTGは意味があったか」までレポート化。
- **24時間／365日稼働のAI業務**：Mac mini複数台でClaude等を常時走らせ、定型業務を並列処理する構想。
- **現場発アイデアの吸い上げ**：AI研修と連動し、現場から出る「こう使いたい」をどんどんAI化する文化づくり。

## 6. データ要件（取得すべきデータ）

理想形として必要なデータ項目。Phase 0 のフォーム設計の出発点とする。

### 6.1 行動データ（日報フォームで取得）★現状欠落

- 日付 / 担当者
- テレアポ件数、訪問件数、銀行訪問件数、DM発送件数
- 有効リード件数（ニーズありそうだった件数）
- 商談ステータスの更新（任意）
- 所感（最小限・任意）

### 6.2 成果データ（既存：契約シート）

- 担当者 / 物件 / 顧客
- 買付日・契約日・決済日（入金完了日）
- 売上額・原価・粗利（買い切り再販）
- ステータス（買付前〜決済）

### 6.3 インセンティブデータ（既存：入出金シート）

- 個人別の成約額・インセンティブ到達額

### 6.4 物件データ（既存：物件概要書／レントロール）

- 固定情報（面積等、変動しない）
- 変動情報（現在の家賃、入居状況、固定資産評価額）

### 6.5 評価ロジック

- 1ページの評価マニュアル（考え方）＝AI考察の判断基準。

## 7. 非機能要件・運用方針

- **使いやすさ**：現場全員が無理なく入力・閲覧できること（Salesforce撤退の教訓）。入力は1日数分で完了する設計。
- **内製性**：外部の大規模システムや高額ライセンスに依存せず、Claude／GAS／Googleフォーム等で内製。
- **運用ルール**：日報入力を一般営業まで含め強制力をもって徹底（入力されなければデータが溜まらず、分析が成立しない）。
- **拡張性**：賃貸管理・福祉事業（8月以降）にも展開できるデータ構造。
- **セキュリティ**：当面は内製で進め、将来セキュリティ要件が高まった段階で外部コモン等の活用を検討（現時点で高年収のシステム専任採用は優先度低と整理）。

## 8. 体制・進め方に関する論点（打ち合わせ補足）

- 当面はAI駆動（Claude等）で佐々木さん主導＋伊藤さんのアドバイスで内製する方針。
- システム専任の高年収採用は、マネジメント不在・成果物の妥当性を判断できない・技術スタック（基幹系：C・COBOL・Java）がモダン開発（Next.js／TypeScript）と乖離、などの理由から現時点では非推奨と整理。
- 現場で実際に営業する人からアイデアが出る環境づくりを重視。AI研修と連動させ、現場発の改善をAI化していく。

## 9. ネクストアクション

1. **【佐々木さん】** 秘密保持契約の締結後、サンプルデータを共有（既存の契約シート・入出金シート・日報サンプル・物件概要書・評価マニュアル）。ファイル授受はチャットワークを使用。
2. **【伊藤さん／サードスコープ】** 受領データをもとに、Phase 1 のダッシュボードUI/UXサンプルを作成。
3. サンプルを見て佐々木さんから追加要望を吸い上げ、そこから逆算して Phase 0 の日報Googleフォームを設計。
4. 日報フォーム移行を実施し、行動量データの蓄積を開始。
5. データが溜まり次第、Phase 2（行動量×成果＋AI考察）へ。
