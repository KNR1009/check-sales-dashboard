# 営業管理ダッシュボード

株式会社サードスコープ「営業管理ダッシュボード化システム 要件定義書（v0.1）」に基づく
**Phase 1 試作（成果・行動量の可視化 ＋ AI考察）** の実装です。
Next.js (App Router) + TypeScript + Tailwind CSS + Recharts で構築しています。

デザインはデジタル庁「ダッシュボードデザインの実践ガイドブック」のカラーパレット・
レイアウト原則・アクセシビリティ要件に準拠しています。

> 表示しているデータはすべて **リアルなダミーデータ**（営業30名・直近12週分を決定論的に生成）です。
> 実運用では Google フォーム（日報）・契約シート・入出金シート由来のデータへ差し替えます。
>
> 行動量は不動産営業の実務に合わせて **7区分**（架電・DM送付・ポスティング・訪問・空き家調査・銀行訪問・追客/反響対応）で記録します。

## セットアップ

```bash
npm install
npm run dev      # 開発サーバー（http://localhost:3000）
npm run build    # 本番ビルド
npm start        # 本番起動
```

## 画面構成

### マネジメントビュー（全員比較）
- **KPIサマリー**：決済売上・契約件数・行動量・全社インセンティブ
- **要対応メンバーのアラート**：「改善必要」「活動不足」を冒頭で炙り出し（要件 1.3）
- **行動量 × 成果の散布図**（★最重要・軸③）：4象限分類。点クリックで個人ビューへ
- **週次推移**：行動量・有効リード・契約件数の12週トレンド
- **個人別インセンティブ**：決済粗利5%で試算した到達額（降順）
- **ランキングテーブル**：行動量・転換率・成果・前週比を一覧

### 個人ビュー（自分の改善点）
- 個人KPI（行動量・有効リード・契約/決済・インセンティブ）
- **AI考察レポート**（軸④）：評価マニュアル（4象限）をルール化した自動考察
- **ネクストアクション**：考察から導いた優先度付きの次の一手（軸④ / F-09）
- 行動量内訳（チーム平均比較）／個人の週次推移
- **フォロー切れ顧客**（14日以上未接触）／パイプライン

## 要件定義書との対応

| 軸 / 機能ID | 要件 | 実装 |
|---|---|---|
| 軸① 行動量 | 架電/DM送付/ポスティング/訪問/空き家調査/銀行訪問/追客・反響対応 ＋ 有効リード（7区分） | KPI・内訳・週次・ランキング |
| 軸② 成果 (F-03) | 契約・決済・売上・粗利・パイプライン | KPI・ランキング・パイプライン |
| 軸③ 相関 (F-07) ★最重要 | 散布図・ランキング・前週比 | 散布図(4象限)・ランキング・前週比列 |
| 軸④ AI考察 (F-08/F-09) | 評価マニュアル組込みの考察・ネクストアクション | `lib/analysis.ts buildInsight` |
| インセンティブ (F-04) | 個人別到達額 | 横棒グラフ・KPI |
| 個人/管理ビュー (3.6) | ビューを分ける | タブ切替 |

## ディレクトリ構成

```
app/
  layout.tsx, page.tsx, globals.css
components/
  Dashboard.tsx           # ビュー切替シェル
  ManagementView.tsx      # マネジメントビュー
  IndividualView.tsx      # 個人ビュー
  ui.tsx                  # 象限バッジ・KPIカード等の共通UI
  charts/                 # Recharts（散布図・週次・インセンティブ）
lib/
  types.ts                # ドメイン型（要件6章対応）
  data.ts                 # ダミーデータ生成（決定論的）
  analysis.ts             # 集計・象限分類・AI考察ロジック
  format.ts               # 表示フォーマッタ
```

## 実データへの差し替え方針（Phase 0 → 本番）

`lib/data.ts` の `generateActivities()` / `generateDeals()` を、
Google スプレッドシート（フォーム回答・契約/入出金シート）の取得処理に置き換えるだけで、
`lib/analysis.ts` 以降の集計・可視化ロジックはそのまま再利用できます。
AI考察（`buildInsight`）は現在ルールベースですが、同じインターフェースで Claude API 呼び出しに差し替え可能です。

## 注記
- 行動量×成果の象限分類は、チーム内の **中央値** を境界に「理想型／効率型／改善必要／活動不足」へ分類しています。
- インセンティブは「決済粗利の5%」の簡易モデルです（実運用では入出金シートの実額に置換）。
- デザイン出典：デジタル庁「ダッシュボードデザインの実践ガイドブック」(2026年3月31日更新)。
